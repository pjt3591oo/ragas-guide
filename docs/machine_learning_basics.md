# 머신러닝 기초 가이드

## 머신러닝이란

머신러닝(Machine Learning)은 명시적으로 프로그래밍하지 않고도 데이터로부터 학습하여
패턴을 찾고 예측을 수행하는 인공지능의 한 분야입니다.

전통적인 프로그래밍에서는 규칙을 직접 코딩하지만, 머신러닝에서는 데이터와 결과를
제공하면 알고리즘이 스스로 규칙을 학습합니다. 이를 통해 스팸 필터링, 이미지 인식,
추천 시스템 등 복잡한 문제를 해결할 수 있습니다.

## 지도학습 (Supervised Learning)

지도학습은 입력(X)과 정답(y)이 쌍으로 주어진 데이터로 학습하는 방법입니다.
모델은 입력과 출력 사이의 관계를 학습하여 새로운 입력에 대한 출력을 예측합니다.

주요 알고리즘:
- 선형 회귀(Linear Regression): 연속적인 값을 예측. 예) 집값 예측
- 로지스틱 회귀(Logistic Regression): 이진 분류. 예) 스팸 여부 판별
- 결정 트리(Decision Tree): 조건 분기로 예측. 해석이 직관적
- 랜덤 포레스트(Random Forest): 여러 결정 트리의 앙상블. 과적합 방지에 효과적
- SVM(Support Vector Machine): 데이터를 분리하는 최적의 초평면을 찾음
- k-최근접 이웃(k-NN): 가장 가까운 k개의 이웃을 기반으로 분류

지도학습의 핵심 과제는 과적합(overfitting) 방지입니다. 훈련 데이터에만 잘 맞고
새로운 데이터에 일반화되지 못하는 현상을 방지하기 위해 교차 검증(cross-validation),
정규화(regularization) 등의 기법을 사용합니다.

## 비지도학습 (Unsupervised Learning)

비지도학습은 정답 레이블 없이 데이터의 구조와 패턴을 발견하는 방법입니다.

주요 알고리즘:
- k-평균 클러스터링(k-Means): 데이터를 k개의 그룹으로 분할
- DBSCAN: 밀도 기반 클러스터링으로 임의 형태의 클러스터 발견
- PCA(주성분 분석): 차원 축소를 통해 데이터의 핵심 특성을 추출
- t-SNE: 고차원 데이터를 2D/3D로 시각화

비지도학습의 일반적인 활용 사례:
- 고객 세분화: 비슷한 행동 패턴의 고객 그룹 발견
- 이상 탐지: 정상 패턴에서 벗어난 데이터 감지 (사기 탐지 등)
- 추천 시스템: 사용자 간 유사도 기반 추천

## 딥러닝 (Deep Learning)

딥러닝은 인공 신경망(Artificial Neural Network)을 사용하는 머신러닝의 하위 분야입니다.
여러 층의 뉴런(노드)이 연결된 네트워크가 데이터에서 계층적 특징을 자동으로 학습합니다.

주요 아키텍처:
- CNN(합성곱 신경망): 이미지 인식, 객체 탐지에 탁월
- RNN(순환 신경망): 시계열 데이터, 자연어 처리에 사용
- Transformer: 어텐션 메커니즘 기반으로 현재 NLP의 표준 아키텍처
- GAN(생성적 적대 신경망): 새로운 데이터(이미지, 텍스트 등)를 생성

딥러닝이 전통적인 머신러닝과 다른 핵심 차이:
- 특성 공학(feature engineering)이 자동화됨
- 대량의 데이터와 GPU 연산 자원이 필요
- 모델의 해석이 어려운 블랙박스 특성

## 모델 평가

머신러닝 모델의 성능을 평가하는 주요 지표:

분류(Classification) 지표:
- 정확도(Accuracy): 전체 중 올바른 예측 비율
- 정밀도(Precision): 양성으로 예측한 것 중 실제 양성 비율
- 재현율(Recall): 실제 양성 중 양성으로 예측한 비율
- F1 점수: 정밀도와 재현율의 조화 평균
- AUC-ROC: 분류기의 전체적인 성능을 곡선 아래 면적으로 표현

회귀(Regression) 지표:
- MSE(평균 제곱 오차): 예측값과 실제값 차이의 제곱 평균
- RMSE(제곱근 평균 제곱 오차): MSE의 제곱근, 원래 단위와 동일
- MAE(평균 절대 오차): 예측값과 실제값 차이의 절대값 평균
- R² 점수: 모델이 설명하는 분산의 비율 (1에 가까울수록 좋음)

교차 검증(Cross-Validation)은 데이터를 k개의 폴드로 나누어 모델을 반복 평가하는
방법으로, 모델의 일반화 성능을 더 정확히 추정할 수 있습니다.

## 데이터 전처리

머신러닝에서 데이터 전처리는 모델 성능에 큰 영향을 미칩니다.

주요 전처리 기법:
- 결측치 처리: 평균/중앙값 대체, 삭제, 또는 모델 기반 보간
- 특성 스케일링: StandardScaler(표준화), MinMaxScaler(정규화)
- 인코딩: 범주형 변수를 수치로 변환 (원-핫 인코딩, 레이블 인코딩)
- 특성 선택: 중요한 특성만 선별하여 차원의 저주(curse of dimensionality) 방지
- 데이터 증강: 학습 데이터를 인위적으로 늘려 과적합 방지 (이미지 회전, 반전 등)

## 실전 워크플로우

일반적인 머신러닝 프로젝트의 진행 순서:

1. 문제 정의: 해결하려는 문제가 분류인지 회귀인지 클러스터링인지 결정
2. 데이터 수집 및 탐색: EDA(탐색적 데이터 분석)로 데이터 이해
3. 데이터 전처리: 결측치 처리, 스케일링, 인코딩
4. 모델 선택 및 학습: 여러 모델을 시도하고 비교
5. 하이퍼파라미터 튜닝: Grid Search, Random Search, Bayesian Optimization
6. 모델 평가: 테스트 데이터로 최종 성능 확인
7. 배포 및 모니터링: 실제 서비스에 적용하고 성능 모니터링
