# 파이썬 기초 가이드

## 변수와 자료형

파이썬은 동적 타이핑 언어로, 변수를 선언할 때 타입을 명시하지 않아도 됩니다.
주요 자료형으로는 int, float, str, bool, list, dict, tuple, set 등이 있습니다.

파이썬에서 모든 것은 객체입니다. 정수 1도 int 클래스의 인스턴스이며,
문자열도 str 클래스의 인스턴스입니다. id() 함수로 객체의 메모리 주소를 확인할 수 있습니다.

## 함수

파이썬에서 함수는 def 키워드로 정의합니다. 함수는 일급 객체(first-class object)로,
변수에 할당하거나 다른 함수의 인자로 전달할 수 있습니다.

lambda 표현식을 사용하면 이름 없는 함수를 한 줄로 작성할 수 있습니다.
주로 map, filter, sorted 등의 고차 함수와 함께 사용됩니다.

## 데코레이터

데코레이터는 다른 함수를 인자로 받아 기능을 확장하는 함수입니다.
@기호를 사용하여 적용합니다. functools.wraps를 사용하면 원래 함수의
이름과 독스트링을 보존할 수 있습니다.

데코레이터의 일반적인 용도:
- 로깅: 함수 호출 시 자동으로 로그 기록
- 캐싱: @functools.lru_cache로 결과 캐싱
- 인증: 웹 프레임워크에서 접근 권한 확인
- 재시도: 실패 시 자동 재시도 로직 추가

## 제너레이터

제너레이터는 yield 키워드를 사용하여 값을 하나씩 생성하는 함수입니다.
메모리 효율적으로 대량의 데이터를 처리할 수 있습니다.

제너레이터 표현식은 리스트 컴프리헨션과 비슷하지만 소괄호를 사용합니다.
(x**2 for x in range(1000000))은 메모리를 거의 사용하지 않습니다.

## GIL (Global Interpreter Lock)

CPython의 GIL은 한 번에 하나의 스레드만 파이썬 바이트코드를
실행하도록 보장하는 뮤텍스입니다. GIL의 주요 특징:

- 메모리 관리를 단순화하여 구현이 쉬움
- CPU 바운드 멀티스레딩의 성능을 제한
- I/O 바운드 작업에서는 GIL이 해제되므로 영향이 적음
- multiprocessing 모듈로 GIL 제한을 우회 가능
- 파이썬 3.13에서 실험적으로 free-threaded 모드 도입

## 비동기 프로그래밍

파이썬 3.5부터 async/await 구문이 도입되었습니다.
asyncio 모듈은 이벤트 루프 기반의 비동기 I/O를 제공합니다.

async def로 코루틴을 정의하고, await로 비동기 작업의 완료를 기다립니다.
여러 코루틴을 동시에 실행하려면 asyncio.gather()를 사용합니다.

비동기 프로그래밍이 유용한 경우:
- 웹 서버에서 다수의 클라이언트 요청 처리
- 여러 API에 동시에 요청 보내기
- 데이터베이스 쿼리 병렬 실행
- 파일 I/O 작업 비동기 처리

## 리스트 컴프리헨션

리스트 컴프리헨션은 기존 리스트로부터 새로운 리스트를 만드는 간결한 방법입니다.
일반적인 for 루프보다 성능이 좋은데, 이는 내부적으로 C 코드로 최적화되어 있기 때문입니다.

딕셔너리 컴프리헨션, 집합 컴프리헨션도 비슷한 문법으로 사용할 수 있습니다.
중첩 컴프리헨션도 가능하지만, 가독성을 위해 2단계 이상은 for 루프를 권장합니다.
